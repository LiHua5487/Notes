
# 基本思想和使用条件

使用条件：**优化原则**，即**整体最优里包含局部最优**

比如给定义一个图，求起点到终点的最短路径，由于最短路径中，任意两点间的距离也一定是最短的，所以可以用 dp 

但是如果把问题换为：求总长模 10 的余数最小的路径，就不满足优化原则，因为总长模 10 最小，不一定代表子路径模 10 最小（比如 A 到 B 有余数为 1 和 6 的两条路径，B 到 C 有余数为 7 的路径，如果在 B 点只保留最小余数 1 ，就会错过 AC 真正的最小余数 3）

---

例（矩阵乘法）：设 $A_1, A_2, \dots, A_n$ 为矩阵序列，$A_i$ 为 $P_{i-1} \times P_i$ 阶矩阵，确定乘法顺序使得计算所有矩阵的相乘结果中，元素相乘的总次数最少
- 输入：向量 $P = <P_0, P_1, \dots, P_n>$ ，代表矩阵大小 

比如 $P = <10, 100, 5, 50>$ ，就代表 $A_1: 10 \times 100, A_2: 100 \times 5, A_3: 5 \times 50$ ，总乘法次数可能为

$$
\begin{aligned}
&(A_1A_2)A_3: 10 \times 100 \times 5 + 10 \times 5 \times 50 = 7500 \\
&A_1(A_2A_3): 10 \times 100 \times 50 + 100 \times 5 \times 50 = 75000
\end{aligned}
$$

考虑 $A_1\cdots A_n$ 的可能的加括号情况数，设为 $f(n)$，最后一次乘法一定是分为左右两部分 $(A_1\cdots A_k)(A_{k+1}\cdots A_n)$ ，可得递推式为 $f(n)=\sum_{k=1}^{n-1}f(k)f(n-k)$ ，这正好是卡特兰数的递推式，得 $f(n)=\frac{1}{n+1} \binom{2n}{n}$ ，状态空间太大了

dp 算法：设 $m[i][j]$ 表示得到 $A_i\cdots A_j$  的最小相乘次数，则递推方程为

$$
m[i, j] =
\begin{cases}
0 & i = j \\
\min_{i \leq k < j} \{m[i, k] + m[k+1, j] + P_{i-1} P_k P_j\} & i < j
\end{cases}
$$

还可以额外搞一个 $s[i][j]$ ，记录次数最小时，$k$ 是多少，这样就能还原出加括号的情况

```
RecurMatrixChain(P, i, j)
1. if (i == j) 
2.     m[i, j] = 0
3.     return 0
4. m[i, j] ← ∞
5. s[i, j] ← i
6. for k ← i to j - 1 do
7.     q ← RecurMatrixChain(P, i, k) 
          + RecurMatrixChain(P, k + 1, j) 
          + p_{i-1} p_k p_j
8.     if q < m[i, j]
9.         then m[i, j] ← q
10.              s[i, j] ← k
11. return m[i, j]
```

复杂性满足递推关系

$$
\begin{aligned}
&T(n) \geq 
\begin{cases}
O(1) & n = 1 \\
\sum_{k=1}^{n-1} (T(k) + T(n-k) + O(1)) & n > 1
\end{cases} \\
&T(n) \geq O(n) + \sum_{k=1}^{n-1} T(k) + \sum_{k=1}^{n-1} T(n-k) = O(n) + 2\sum_{k=1}^{n-1} T(k)
\end{aligned}
$$

数学归纳法证明 $T(n) \geq 2^{n-1}$  
- $n=2$, 显然为真  
- 假设对于任何小于 $n$ 的 $k$ 命题为真，则

$$
\begin{aligned}
T(n) &\geq O(n) + 2\sum_{k=1}^{n-1} T(k) \geq O(n) + 2\sum_{k=1}^{n-1} 2^{k-1}  \\
&= O(n) + 2(2^{n-1} - 1) \geq 2^{n-1}
\end{aligned}
$$

这还是太慢了，这是因为 $m[i, j]$ 在不同的递归进程中，被重复计算了很多次 ，实际上，只需要在函数的一开始加上下面这段就行

```
if m[i][j] != -1 // m[i, j] 在一开始被初始化为 -1
	then return m[i][j]
```

也可以改成非递归形式，这就更贴近 dp 了

```
MatrixChain(P, n)
1. 令所有的 m[i, i] 初值为 0
2. for r ← 2 to n do // 遍历矩阵链长 r 
3.     for i ← 1 to n - r + 1 do // 遍历起始位置 i 
4.         j ← i + r - 1 // 计算终止位置 j 
5.         m[i, j] ← m[i+1, j] + pi-1 * pi * pj // Ai(Ai+1..Aj)
6.         s[i, j] ← i
7.         for k ← i+1 to j-1 do // 遍历分割位置 k 
8.             t ← m[i, k] + m[k+1, j] + pi-1 * pk * pj
9.             if t < m[i, j]
10.                 then m[i, j] ← t
11.                      s[i, j] ← k // (Ai..Ak)(Ak+1..Aj)
```

时间复杂度为 $O(n^3)$ 

# 设计步骤

- 状态是什么（比如 $d[i][j]$ 取什么含义）
- 状态转移方程
- 边界条件（最小子问题的答案，比如 $d[i][i]=0$）
- 计算顺序
- 要不要记录方案（比如矩阵乘法里的 $s[i][j]$）

# 应用实例

## 投资问题

- $m$ 元钱，$n$ 项投资，$f_i(x)$ 是将 $x$ 元投入第 $i$ 项项目的效益
- 目标函数 $\max \{f_1(x_1) + f_2(x_2) + \dots + f_n(x_n)\}$ 
- 约束条件 $x_1 + x_2 + \dots + x_n = m$，$x_i \in \mathbb{N}$ 

设 $F_k(x)$ 表示 $x$ 元钱投给前 $k$ 个项目的最大效益，则递推方程和边界条件为  

$$
\begin{aligned}
&F_k(x) = \max_{0 \leq x_k \leq x} \{f_k(x_k) + F_{k-1}(x - x_k)\} \quad k > 1 \quad \\
&F_1(x) = f_1(x)
\end{aligned}
$$

表中有 $m$ 行 $n$ 列，共 $mn$ 项，其中 $F_k(x)$ 项 $(2 \leq k \leq n, 1 \leq x \leq m)$ 需要 $x+1$ 次加法，$x$ 次比较  
- 加法次数：$\sum_{k=2}^{n} \sum_{x=1}^{m} (x+1) = \frac{1}{2}(n-1)m(m+3)$ 
- 比较次数：$\sum_{k=2}^{n} \sum_{x=1}^{m} x = \frac{1}{2}(n-1)m(m+1)$ 
- 总时间复杂度：$W(n) = O(nm^2)$ 

## 最长公共子序列 LCS

给定序列 $X=<x_1,\cdots,x_m>,Y=<y_1,\cdots,y_n>$ ，求二者的最长公共子序列（不是子串，不要求连续）

设 $Z=<z_1,\cdots,z_k>$ 为 $X_m$ 和 $Y_n$ 的 LCS
- 若 $x_m=y_n$ ，则 $z_k=x_m=y_n$ ，且 $Z_{k-1}$ 是 $X_{m-1}$ 与 $Y_{n-1}$ 的 LCS
- 若 $x_m \neq y_n, z_k \neq x_m$ ，则 $Z_k$ 是 $X_{m-1}$ 与 $Y_n$ 的 LCS
- 若 $x_m \neq y_n, z_k \neq y_n$ ，则 $Z_k$ 是 $X_m$ 与 $Y_{n-1}$ 的 LCS

考虑 $X$ 与 $Y$ 的子序列 $X_i = <x_1, x_2, \dots, x_i>$，$Y_j = <y_1, y_2, \dots, y_j>$ 
- $C[i,j]$ 为 $X_i$ 与 $Y_j$ 的 LCS 的最大长度，可得递推方程为

$$
C[i,j] = 
\begin{cases} 
0 & 若 i = 0 或 j = 0 \\ 
C[i-1,j-1] + 1 & 若 i,j > 0, x_i = y_j \\ 
\max\{C[i,j-1], C[i-1,j]\} & 若 i,j > 0, x_i \neq y_j 
\end{cases}
$$

- $B[i,j]$ 为标记函数，值为字符 $\nwarrow,\leftarrow,\uparrow$，分别表示 $C[i,j]$ 取得最大值时的三种情况

```
LCS(X,Y,m,n)  
1. for i←1 to m do  
2.   C[i,0]←0  
3. for i←1 to n do  
4.   C[0,i]←0  
5. for i←1 to m do  
6.   for j←1 to n do  
7.     if X[i]=Y[j]  
8.       then C[i,j]←C[i-1,j-1]+1  
9.       B[i,j]←'↖'  
10.     else if C[i-1,j]≥C[i,j-1]  
11.       then C[i,j]←C[i-1,j]  
12.       B[i,j]←'↑'  
13.     else C[i,j]←C[i,j-1]  
14.       B[i,j]←'←'
```

```
Structure Sequence(B, i, j) // 根据 B 还原解
1. if i=0 or j=0 then return //一个序列为空  
2. if B[i,j] = '↖'  
3.   then 输出 X[i]  
4.   Structure Sequence(B, i-1, j-1)  
5. else if B[i,j]= '↑' then Structure Sequence(B, i-1, j)  
6.   else Structure Sequence(B, i, j-1)  
```

算法的复杂度  
- 计算优化函数和标记函数：$O(mn)$   
- 构造解：每一步至少缩小 X 或 Y 的长度，时间 $\Theta(m+n)$ 
- 空间：$\Theta(mn)$
