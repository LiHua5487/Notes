# Learning Frameworks

## 课程学习 Curriculum Learning

核心思想：模仿人类的学习过程，让模型从简单到复杂地学习任务，而不是一开始就面对最困难的问题，这种循序渐进的方式可以帮助模型更稳定、更快地收敛，避免一开始就陷入糟糕的局部最优解，通常能获得更强的泛化能力

比如在训练机器狗走路时，可以先在平地上训练，而后再加入各种复杂地形

## 分层学习 Hierarchical Learning

核心思想：对于需要多步决策的长周期任务（尤其是 RL），直接学习从感知到动作的映射非常困难，可以将复杂的任务分解成多个层次的子任务，每个任务采用一个模块去解决，使学习更高效、策略更可解释，也更容易复用学到的子技能

比如在加入视觉的机器狗中，可以分为以下层次
- 感知层 Perception Module ：通过视觉模块处理从环境中获得的原始信息（如点云数据），并输出一个对周围环境的高质量理解，使系统能够识别出地形的主要特征（如平坦面、障碍物、攀爬点等）
- 导航层 Navigation Module ：根据视觉模块提供的特征，规划一条行动路线，同时进行避障，并决定在什么位置要用什么动作，频率较低
- 运动层 Locomotion Module ：智能体的技能库，包含一系列动作，负责控制动作的具体实现（如每个关节要怎么移动），需要高频决策

## 特权学习 Privileged Learning

核心思想：在训练阶段，老师可以利用一些额外信息（称为特权信息）来指导学生模型，让模型在测试时仅凭普通输入也能达到接近使用特权信息的性能

![[EAI导论/imgs/img11/image-2.png]]

---

Visual Policy ：输入视觉信息，能直接输出要进行什么动作，相比于上面的分层学习，这是端到端的

其模型结构分为两部分
- Visual Encoder ：从视觉输入提取出一个特征向量，代表对 state 的预测
- 策略网络：一般是 MLP ，有时也会加入循环网络（如LSTM）来处理时序信息，负责根据视觉特征输出动作

但其训练是很困难的，一个原因是反向传播时的 Policy Gradient 的噪声可能对 Vision Encoder 的学习造成破坏，导致其训不好；可以先对 encoder 进行预训练（如使用 ImageNet），而后把它给 freeze 住，反传时不更新其参数，但 encoder 提供的信息可能本来就不够，导致后面 MLP 的表现上不去

为了解决 encoder 提供的信息不足的问题，可以在预训练时把动作信息也作为一种模态加入到训练过程中；对于 MLP 的表现问题，可以提供特权信息（训练时有的额外信息，测试和实际应用时没有），把一些需要视觉提取的 state 信息（比如障碍物的位置），用 gt （一般直接从模拟器获取）直接给出，先训练出一个 state-based policy （没有视觉观测，因为 state 是直接给到的），而后只需要训练一个预测这些 state 的网络就行了，但这很考验 state-based policy 对于提供的 state 的误差的鲁棒性

---

**策略蒸馏 Policy Distillation** ：利用特权学习，把上述的 state-based policy 作为教师策略，而对于学生策略，它是 vision-based，只能根据观测 o 去决策动作 a 

在训练时，对于某个状态 s ，学生策略根据观测结果进行决策，产生动作 a ，而教师策略直接根据 s 决策出动作 a* ，把 a* 作为监督信号，从而使学生模型学习出一个从观测 o 到动作 a 的映射，可见这是一种监督学习（varience 相比 RL 要小），这个过程也被称为 DAgger （与之前 Distribution Shift 中的相比，相当于把人类的标注换成了教师策略的监督）

当然，这么做的前提是视觉观测得到的信息和状态信息的差距不会太大，即教师不会太超模，学生能获取的信息也要足够

考虑一个场景，我们要训练一个机器狗跳过一个沟，这需要其在空中时（使用 base height 衡量其距地面的高度）能够做出准备落地的动作，那视觉观测中就也应该能看到地面并预测出相应信息

>**知识蒸馏 Knowledge Distillation** ：以图像分类为例，假设现在有一个训练好的，但是比较庞大的模型（如 ResNet151），它很难部署到一些轻量设备上，我们想搞一个轻量级的模型（如 ResNet25），但如果直接训练，使用 one-hot 向量作为 ground truth ，这个小模型很难学的很好，毕竟学习能力有限，但我们可以把那个大模型的输出作为 gt ，而这个向量不是 one-hot 的，这就相对容易学习了
>比如对于两个比较相近的事物，one-hot 的 gt 可谓天差地别，但是大模型的输出就天然包含这种相似性（两个概率相近），同时也会告诉小模型哪个是正确的（概率更大）
>这么训练的结果虽然比不上大模型，但往往比直接拿 one-hot 训的效果要好

由于是把教师策略的输出当作监督信号，所以学生一般不会有青出于蓝而胜于蓝的现象，即天花板水平一般就是教师水平，而且可能继承教师策略的一些缺陷

---

**Asymmetric Actor-Critic (AAC)**：actor 需要部署到现实世界，但是 critic 只是辅助学习的，完全可以让 critic 获取一些特权信息

# Loco-Manipulation

**Loco-Manipulation** ：任务中同时包含移动和操作（比如走过去抓东西）

比如一个机器狗上面有个机械爪，我们训好了夹爪怎么抓，也能训机器狗怎么走路，但关键在于走到什么位置、调整到什么姿态，最方便抓取，可以对此设一个 reward 训练机器狗的移动与姿态

相比于机器狗这种四足 quadrupedal ，人形机器人这种两足 bipedal 的训练起来往往会更加困难

![[EAI导论/imgs/img11/image-4.png]]

双足机器人常常使用**动作捕捉 (Motion Capture, Mocap)** 来获得训练数据，而动捕的人和机器人的体态不一样，可以采用 Motion Retargeting 将从人身上采集到的动作映射到机器人上

目前为止，设定的 goal 基本上都是 train time 的，即在训练时就已经定下来目标了，这就导致训练好的模型往往只能干这一个事；而 test time 的 goal 则是在测试时设置一个目标（可能和训练时不一样），看机器人表现得怎么样
