# 简介

本项目旨在深入探究图像生成模型的可控性，以 SIGGRAPH 2023 的 DragGAN 工作作为基础，该工作允许用户通过拖拽图像中的点来直观地操控物体姿态、形状等属性，实现较为精确的控制

项目目标：复现 DragGAN 论文中的定性结果，并实现追踪鲁棒性增强的改进

总体方案
- 部署 DragGAN 官方仓库，理解其运动监督和点追踪机制，复现论文中的定性结果，包括与 UserControllableLT 方法的比对
- 尝试用多尺度特征、运动轨迹约束、光流估计、粒子追踪器等方法来解决原始方法在纹理稀疏区域的追踪漂移问题，并通过对比实验验证效果

# 问题陈述

DragGAN 在纹理稀疏区域进行点追踪时效果不佳，易出现漂移问题，主要原因如下
- 特征模糊：无纹理区域中的像素在特征空间难以彼此分辨，导致最近邻搜索无法准确定位
- 缺乏时间一致性：每次迭代独立搜索，未能利用运动轨迹的平滑性等约束，导致盲目匹配

数据集
- 复现阶段：我们将使用 DragGAN 论文中涉及的图像来检验复现结果
- 改进阶段：除了复现阶段的图像，我们将额外收集一些包含较大面积的纹理稀疏区域的图像，与原始 DragGAN 进行对比，以验证改进是否有效

预期结果
- 成功运行 DragGAN 官方代码，复现论文中展示的交互编辑效果
- 实现改进的追踪机制，并在纹理稀疏场景下获得比原始最近邻搜索更稳定、更准确的点追踪结果

评估标准
- 定性：视觉对比编辑后的图像内容、点的追踪轨迹是否合理
- 定量：使用追踪点的位置误差（如最终点与目标点的 L2 距离）衡量点追踪的准确性，并对比图像的 FID (Frechet Inception Distance) 整体评价图像编辑的质量

# 技术方法

复现阶段
- 环境配置：鉴于本地环境与官方仓库的运行环境可能不同，需要进行 python 环境配置与代码的调整
- 代码分析：定位代码片段，与论文内容对照，逐行分析
- 复现结果：运行 DragGAN 图形界面程序，对图像进行编辑；部署 UserControllableLT ，与 DragGAN 的编辑结果进行对比

改进阶段
- 多尺度特征融合追踪：不只采用某一层的特征图，而是采用多层的特征图，将其拼接或加权融合，而后进行最近邻搜索
- 引入运动轨迹约束：使用一个简单的运动模型（如匀速直线运动）来预测下一帧中控制点的位置，在预测位置附近进行最近邻匹配，而非在原位置附近；对于多个候选匹配点，可结合历史轨迹，衡量轨迹平滑性来进行选择
- RAFT 光流估计：对比前后两帧间的光流，从而更新控制点的位置
- PIPs 粒子追踪器：部署 PIPs 粒子追踪器预测控制点位置
