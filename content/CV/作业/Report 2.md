
# Harris Corner Detection

可得角点检测中的响应函数为
$$\theta = g(I_x^2)g(I_y^2) - [g(I_xI_y)]^2 - \alpha[g(I_x^2) + g(I_y^2)]^2$$
为此，先计算出图像的 $x$ 和 $y$ 方向的梯度 $I_x$ 和 $I_y$ ，并进一步计算 $I_{xx}=I_x\cdot I_x,\ I_{yy}=I_y\cdot I_y,\ I_{xy}=I_x\cdot I_y$ ，而后对三者应用高斯模糊，最后代入公式计算响应函数，根据阈值进行筛选，额外应用 `corner_selection` 函数保证一个小范围内只有一个角点

# RANSAC

计算单应矩阵时，先随机选取 4 个点计算出一个单应矩阵，据此计算每个点变换后的坐标，并计算与对应点坐标之间的误差，把误差小于一定阈值的视为 inlier ，选择 inlier 比例最大的情况，用这些 inlier 计算单应矩阵

# Pair-image Stitching

先对两个图像进行角点检测，计算角点所在处的梯度直方图，先统计出主方向，再分别统计每个细分的小区域内的直方图，把梯度模长累加到对应区间，由于边缘处对于特征的贡献比较小，可以用一个高斯函数对梯度模长进行加权，最后把直方图统计结果排列成一个向量，并归一化，作为这个点的特征，并进行特征点匹配，可得以下结果

# Panorama generation

生成全景图时，依次进行图片的特征检测、对应点匹配与单应矩阵计算，并拼贴到一起

这里我对原来的全景图生成算法进行了改进，原来的算法逻辑是：以中间的图片为基准，把左右两侧的图片拼接到上面，在检测对应点时，是将待拼接图像与现有的拼接结果进行匹配

这有一个问题，就是拼接结果上，两侧的图像经过了非线性变换，其特征被改变了，且越靠近边缘这种变形越严重，这就导致全景图生成范围受限

为此，可以将待拼接图像与边缘处的图像（拼接前）进行匹配并计算出单应矩阵 $H_1$ ，又已知边缘处的图像拼接到全景图上时需要经过变换 $H_0$ ，那待拼接图像需要进行的变换就是 $H_0H_1$

# Analyze

将特征匹配部分改为使用 SIFT feature ，可以发现得到的对应点数量多了很多，说明 SIFT 能检测出更多的特征，描述性更强

![[CV/作业/imgs/img2/image-1.png]]

使用 SIFT feature 得到的对应点进行全景图生成，可以扩大允许的视野，因为原本使用 Harris Corner 时，可能一个图片明明和中间的图片有匹配的部分，但可能并不是角点，导致无法匹配或者匹配数量太少了，全景图生成终止，而使用 SIFT 就能将这些特征点检测出来，且由于数量更多，计算结果更稳定

比如在 Xue-Mountain-Entrance 的全景图生成中，使用 Harris Corner 就只能对前 5 个图像进行生成，而使用 SIFT 后扩大到 7 个图片，只不过这个拉伸太离谱了
