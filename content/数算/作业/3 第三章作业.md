
# 1

用顺序表实现一个栈，声明一个名为 `Stack` 的结构体，其含一个指向顺序表起始地址的指针 `ps` 和一个栈顶下标 `top` ，为了能获取最小值，还需要一个属性 `min` 记录最小值，但是如果弹出的元素是最小值，就无法得知剩余部分的最小值了

为此，考虑对最小值进行编码，当新的最小值 `x_min` 入栈时，不储存其本身，而是储存 `2 * x_min - min` （注意这里的 `min` 是入栈前的），再更新 `min` 为 `x_min` ，这个编码值是小于更新后的 `min` 的，而且可以从编码值恢复出其入栈前的 `min`

```cpp
// push
top++;
if (x < min) {
	ps[top] = 2 * x - min;
	min = x;
} else {
	ps[top] = x;
}

// pop
if (ps[top] < min) {
	out = min;
	min = 2 * min - ps[top];
} else {
	out = ps[top]
}
top--;
return out;

// top
return ps[top];

// getMin
return min;
```

# 2

每一种出栈序列都一一对应一种入栈与出栈操作的排序，而在这一系列操作中的任意时刻，入栈次数需要不少于出栈次数，这可以转化为一个经典模型：沿网格线从原点走到 $(n,n)$ ，每次都只能往上或往右走一格，且不越过 $y=x$ （即右走次数不少于上走次数），求移动方式

假设没有禁止触碰的限制，总走法数为

$$N_{\text{all}} =C_{2n}^n$$

对于越过 $y=x$ 的走法，其必与 $y = x+1$ 相交（因为在超越点一定是向上走的），把从这个交点到终点 $(n,n)$ 的路径对称过去，就会发现每条不合法的路径都一一对应一条从原点到 $(n-1,n+1)$ 的路径，而这个方法数是 

$$N_{\text{invalid}}=C_{2n}^{n-1}$$

那么合法的走法数就是 

$$N_{\text{valid}}=C_{2n}^n-C_{2n}^{n-1}$$ 

# 3

## 3.1

撤销和恢复的伪代码如下

```cpp
// undo
if (S1.empty()) return;
operation = S1.pop();
S2.push(operation);

// redo
if (S2.empty()) return;
operation = S2.pop();
S1.push(operation);
```

## 3.2

执行 `do A, do B, do C` 后， `S1: A, B, C; S2: _` （左为栈底，右为栈顶， `_` 代表栈空）
执行 `undo` 后， `S1: A, B; S2: C`
执行 `do D` 后， `S1: A, B, D; S2: C`
执行 `undo` 后， `S1: A, B; S2: C, D`
执行 `redo` 后， `S1: A, B, D; S2: C`

最终 `S1: A, B, D; S2: C`

# 4

## 4.1

假设把 Q1 当作主要存储与操作对象，把 Q2 当作辅助的队列，初始为空，对于 `push` ，我们希望在弹出时能弹出最后进入的元素，但队列是先入先出的，所以需要将数据逆序存储，为此，先把 Q1 中元素全放到 Q2 ，再把 `x` 加入到 Q1 ，而后把 Q2 的元素放回 Q1 ，这样 Q1 里的元素就是逆序的了，在弹出和判空时，只需要对 Q1 执行相应操作即可

```cpp
// push
while (!isEmpty(Q1)) {
	item = deQueue(Q1);
	enQueue(Q2, item);
}
enQueue(Q1, x);
while (!isEmpty(Q2)) {
	item = deQueue(Q2);
	enQueue(Q1, item);
}

// pop
return deQueue(Q1);

// isEmpty
return isEmpty(Q1);
```

## 4.2

`push` 中进行了 2 次队列转移与 1 次入队，每次转移需要遍历队列元素，而入队出队时间复杂度为 $O(1)$ ，所以总时间复杂度是 $O(n)$

`pop` 和 `isEmpty` 中只进行 1 次操作，时间复杂度是 $O(1)$

## 4.3

`push(1)` 时， `Q1: 1; Q2: _` （左为队尾，右为队头）
`push(2)` 时， Q1 元素先转移到 Q2 ，此时 `Q1: _; Q2: 1` ，而后 2 入队，此时 `Q1: 2; Q2: 1` ，最后 Q2 元素转移回 Q1 ，此时 `Q1: 1 2; Q2: _`
`push(3)` 时同理，最后 `Q1: 1 2 3; Q2: _`
`pop` 时 Q1 队头出队，弹出 3 ，而后 `Q1: 1 2; Q2: _`
后续过程同理

最终 `pop` 先后顺序为 `3, 4, 2`
