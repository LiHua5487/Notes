
# 1 Parallel Coordinate

Parallel Coordinate 可视化方法就是把多个垂直平行的坐标轴并列排列，每个轴代表一个数据维度，用一条折线连接所有轴上的对应值

在提供的示例 [Protovis - Parallel Coordinates](https://mbostock.github.io/protovis/ex/cars.html) 中，还实现了下面的交互
- 每个坐标轴有一个滑块，代表选中的范围，范围外的线条将变成灰色
- 鼠标在距离坐标轴一定范围内，且未触碰滑块时，可以通过框选来设置这个坐标轴的滑块范围，单击则会重置为最大范围
- 鼠标触碰到滑块时，可以通过拖拽来上下移动滑块，调整范围

这里有一些细节
- 框选与移动滑块是互斥的，即处在一种交互下时，不会检测另一种交互
- 无论鼠标移动到哪，当前处理的坐标轴在交互开始时就定下来了，不会切换

我也对这些交互进行了基本的实现（不一定完全一样），但是**受限于鼠标事件监听的 API**，可能会出现一些细节问题，包括但可能不限于
- 框选时，滑块上界可能会突然窜到鼠标高度
- 处理一个坐标轴时，如果鼠标移动到别的坐标轴，可能跳到另一个坐标轴

注意，这些问题的原因来自于**项目提供的鼠标事件监听的 API 不完整**，而非代码的逻辑问题，实际上，交互事件的起始与终止的判断依据是**鼠标是否按下**，但 API 只提供了单击（同一位置按下和释放）和拖拽（按下并移动）的监听

# 2 LIC

LIC 沿着流线方向对噪声纹理进行卷积，从而生成能反映向量场结构的图像（好比在一个磁场上随机撒一堆铁屑）
- 从每个像素出发，沿着向量场方向（正向和反向）移动一定步数
- 沿着轨迹对输入的噪声纹理值进行加权平均，这里我参考示例代码，采用 $\cos^2$ 函数加权（离得越远，权重越小）
