## (1)

由于表中元素升序排序，所以可以比较 $p$ 处的 key 和待查找的 key 的大小，决定往前驱方向找，还是往后继方向找，然后依次比较即可

```cpp
Node* search(Node* head, Node* &p, int K) {
    if (!head) return nullptr; // 表空，就不用找了
    Node* curr = p ? p : head; // p空，就从头开始
    
    if (curr->key == K) {
        p = curr;
        return curr;
    }

    if (K < curr->key) {
        Node* tmp = curr->prev;
        while (tmp && tmp->key > K) {
            tmp = tmp->prev;
        }
        if (tmp && tmp->key == K) {
            p = tmp;
            return tmp;
        }
        return nullptr;
    } else {
        Node* tmp = curr->next;
        while (tmp && tmp->key < K) {
            tmp = tmp->next;
        }
        if (tmp && tmp->key == K) {
            p = tmp;
            return tmp;
        }
        return nullptr;
    }
}
```

## (2)

设 $p$ 指向表中第 $i$ 个节点，待查找的节点在第 $j$ 个，则检索长度为 $L(i,j)=\lvert i-j \rvert+1$ 
由于 $i$ 和 $j$ 是随机选取的，则期望检索长度为

$$
E[L] = \frac{1}{n^2} \sum_{i=1}^n \sum_{j=1}^n (|i - j| + 1)
$$

把这个求和式分成两部分，可得

$$
\sum_{i=1}^n \sum_{j=1}^n |i - j| = \frac{n(n+1)(n-1)}{3}, \quad \sum_{i=1}^n \sum_{j=1}^n 1 = n^2
$$

则期望为

$$
E[L] = \frac{\frac{n(n+1)(n-1)}{3} + n^2}{n^2} = \frac{n^2 - 1}{3n} + 1 = \frac{n}{3} + 1 - \frac{1}{3n}
$$

当 $n$ 很大时，$\frac{1}{n} \to 0$ ，期望近似为

$$
E[L]=\frac{n}{3} + 1
$$

遍历 $S_2$ 中的每个元素 $s$ ，计算 $target-s$ ，在 $S_1$ 里查找有没有这个数；可以根据 $S_1$ 建立一个索引表加速查找

```
map = {} // 将S1的元素映射到其出现的位置
for i in range(N):
	map[S1[i]].append(i) // 若键不存在，则默认将其值初始化为空表
res = [] // 存储结果
for j in range(M):
	if !map[target - S2[j]].empty():
		for i in map[target - S2[j]]:
			res.append((i, j))
```

- 构建 map 的时间复杂度为 $O(N)$ 
- 假设最后得到的索引对总数为 $R$ ，则遍历查找的时间复杂度为 $O(M+R)$ 
- 总时间复杂度为 $O(N+M+R)=O(N+R)$ （因为 $M=O(\log N)$ ）
