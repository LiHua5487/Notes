
我使用了 pytorch 的 `DistributedDataParallel(DDP)` 来实现数据并行，在服务器运行了测试程序

# 运行方式

直接运行 `task2.py` 即可，我没有使用 `argparse` ，因为懒得打那么多后缀

# 结果分析

使用 resnet34 模型（经过调整以适配 CIFAR10），优化器为 AdamW，超参数设置如下

```
BATCH_SIZE = 128
LR = 0.001
WEIGHT_DECAY = 5e-4
EPOCHS = 10
```

最终的对比结果如下

```
DDP训练（使用4GPU）
Best Acc: 83.92%  总耗时: 161.20秒
单GPU训练
Best Acc: 84.72%  总耗时: 229.51秒
```

可见在一定情况下，使用数据并行可以提高训练效率，但是加速的倍率并不等于 GPU 的数量，可能是因为 GPU 通信过程无法完美与计算过程同时进行，需要等待与同步
